## 技术栈

### 前端框架

- **React**
- **TypeScript**
- **Vite**

### UI与样式

- **Tailwind CSS **
- **Ant Design**
- **qweather-icons(和风天气图标)**

### 状态管理与路由

- **Zustand**：
- **React Router DOM**

### 数据可视化

- **Chart.js 4.5.1**
- **react-chartjs-2 5.3.1**

### 工具库

- **pinyin-pro 3.28.0**

### 后端模拟

- **JSON Server**

## 使用
1. 打开根目录在终端输入 npm run server 启动 JSON Serve(weatherSearch/mock/db.json)
2. 接着输入 (pnpm dev)/(npm run dev) + o + enter打开浏览器页面

# 小记

基础要求
1.用户体系

- 支持游客模式直接使用，或完成注册/登录(√)
- 登录后支持收藏城市（本地存储/模拟接口），下次登录自动加载收藏列表
2.核心查询功能
- 城市搜索与选择：支持城市名称/拼音搜索，下拉联想匹配；预设热门城市快捷入口（如北京、上海、广州）
- 多维度天气数据获取：(√)
  - 实时天气：温度、湿度、风力、风向、气压、能见度、空气质量
  - 预报数据：7天预报（日期、最高/最低温、天气状况）
  - 生活指数：穿衣、运动、洗车、紫外线等建议
3. 数据可视化与交互体验
- 天气卡片展示：实时天气采用卡片式布局，包含天气图标（晴/雨/云等）(√)
- 预报列表：用纵向/横向列表展示7天预报，24小时预报采用时间轴展示(√)
- 交互反馈：
  - 切换城市时展示加载状态（骨架屏/loading 动画）
  - 天气图标随昼夜/天气类型动态变化（如可以添加天气图标动画）
  - 提供刷新按钮手动更新天气数据
4. 页面基础能力
- 响应式适配：适配 PC 端（≥1200px）、移动端（<768px）布局
- 错误处理：
  - 网络异常时提示用户检查网络
  - 城市搜索无结果时展示空状态(√)
  - API 接口报错时给出提示（如“数据加载失败，请稍后重试”）
- 本地存储：
  - 自动保存最近查询的城市列表，页面刷新后恢复
  - 收藏城市列表持久化存储（localStorage）
  进阶

3. 主题切换(切换白天/夜晚模式)
